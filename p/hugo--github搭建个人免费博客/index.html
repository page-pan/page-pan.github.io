Hugo 创建本地站点

步骤1：安装 Hugo

下载hugo

https://github.com/gohugoio/hugo/

点击tags，选择一个较新稳定版本（v0.155.2），下载后解压到一个目录；

如：D:\blog\hugo155

hugo.exe

LICENSE

README.md

三个文件；

步骤2：创建站点

直接到hugo解压的目录（hugo.exe）所在目录。

hugo new site mysite

Congratulations! Your new Hugo site was created in D:\blog\hugo155\mysite.

Just a few more steps...

1. Change the current directory to D:\blog\hugo155\mysite.
2. Create or install a theme:
   - Create a new theme with the command "hugo new theme <THEMENAME>"
   - Or, install a theme from https://themes.gohugo.io/
3. Edit hugo.toml, setting the "theme" property to the theme name.
4. Create new content with the command "hugo new content <SECTIONNAME>\<FILENAME>.<FORMAT>".
5. Start the embedded web server with the command "hugo server --buildDrafts".

根据上面说明就可以创建一个web静态站点。

步骤3：添加主题与配置

  1. 到对应目录 cd mysite

  2. 安装一个主题，

    到hugo官网 The world’s fastest framework for building websites https://gohugo.io/

    点击Schemes，选择stack主题，点击下载 到 https://github.com/CaiJimmy/hugo-theme-stack 点击tags，选择一个稳定版本

    （hugo-theme-stack-3.34.1.zip），放到mysite的themes目录下，解压改名如hugo-theme-stack-3.34.1，改名hugo-theme-stack，并将其中的demo目录中的content和hugo.yaml拷贝到新建的站点目录（mysite）下，mysite\content\post目录下rich-content这个目录删除（引用国外yutube等网站视频，在国内导致无法启动）。

  3. 使用hugo.yaml替换hugo.toml

    mysite目录下有一个hugo.toml和hugo.yaml重复（功能一样，使用hugo.yaml），需要删除hugo.toml。

    此时就可以启动测试

    hugo.exe server -D

    默认：http://localhost:1313/

    修改适配

    修改hugo.yaml去掉不必要的语言等信息；

    缺省语音更换成中文 DefaultContentLanguage: zh-cn

    hasCJKLanguage: false ==》 hasCJKLanguage: true

    dateFormat:

    ​ published: 2006-01-02

name: deploy

# 代码提交到main分支时触发github action
on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
        - name: Checkout
          uses: actions/checkout@v4
          with:
              fetch-depth: 0

        - name: Setup Hugo
          uses: peaceiris/actions-hugo@v3
          with:
              hugo-version: "latest"
              extended: true

        - name: Build Web
          run: hugo -D

        - name: Deploy Web
          uses: peaceiris/actions-gh-pages@v4
          with:
              PERSONAL_TOKEN: ${{ secrets.BLOGTOKEN }}
              EXTERNAL_REPOSITORY: YourGitHubName/blog
              PUBLISH_BRANCH: main
              PUBLISH_DIR: ./public
              commit_message: auto deploy

BLOGTOKEN为创建的https://github.com/{youName}/mysite.git的action变量（存储访问所有仓库的token)

https://github.com/settings/ 最下面 Developer Settings Person access tokens 创建一个 Tokens (classic)： 记住 token

仓库https://github.com/{youName}/mysite.git setting下 Securits and Variables 创建名字：BLOGTOKEN，值是前面的token内容。

创建新的博客文章，再次提交并推送到github时，会自动构建部署主页更新。
