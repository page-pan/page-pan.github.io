AI 幻觉（Hallucination）规避方法

2026-03-14
阅读时长: 8 分钟

AI 幻觉：大模型编造事实、引用不存在文献、逻辑自洽但错误、答非所问却很自信。

一、最立竿见影：Prompt 工程

  1. 强制 “不知道就说不知道”

    请严格遵守：

    1. 如果你不确定、没有把握、信息不足，直接说“我不确定”，不要编造。
    2. 不要生成不存在的人名、数据、文献、链接、政策文件。
    3. 只输出你确定为真的内容。

  2. 要求标注来源

    回答时必须：

    • 引用可验证来源
    • 不确定来源时明确标注“无可靠来源”
    • 禁止虚构来源、虚构引用

  3. 分步思考 + 自我校验（COT + Self-Correction）

    请按步骤：

    1. 先提取问题关键点
    2. 检索/回忆可靠知识
    3. 组织答案
    4. 自我检查：是否有编造？是否有矛盾？是否有不确定内容？
    5. 只输出最终正确答案

  4. 限制输出范围

    • 只回答某领域
    • 只基于提供的文档回答
    • 禁止扩展、禁止联想

二、最强治本方案：RAG（检索增强生成）

RAG 是目前工业界抑制幻觉最有效手段。

核心逻辑

不依赖模型 “记忆”，而是：

检索真实文档 → 让模型只基于文档回答 → 禁止模型自由发挥

落地要点

  1. 高质量知识库

    • 权威：官方文档、论文、标准、手册、财报、法规
    • 去噪：清洗错误、重复、过时信息

  2. 精准检索

    • 向量库 + 关键词混合检索
    • 召回 Top3~5 最相关片段

  3. 强约束 Prompt

    请只根据下面提供的资料回答， 资料里没有的内容，不要回答，不要编造。

  4. 引用溯源

    回答末尾自动附上：参考资料：xxx 第x段

三、模型层面：从源头减少幻觉

  1. 选择更稳定的模型
  • 闭源：GPT-4、Claude 3、Gemini Advanced 幻觉显著更低
  • 开源：Llama 3、Qwen、GLM 系列新版本 > 小模型 / 旧模型
  1. 微调策略（Fine-tuning）
  • SFT（有监督微调）：用高质量、事实准确的问答对
  • 偏好对齐（RLHF/DPO）：强化 “诚实”、“拒绝编造”
  • 禁止用低质量爬虫数据、混杂错误数据微调
  1. 温度（Temperature）调低
  • 生成更保守、更确定
  • 建议：0~0.3（ factual 任务）
  • 创意任务可适当调高

四、架构层面：外挂工具，杜绝瞎编

让模型不能编，只能查。

  1. 联网搜索插件
    • 实时拉取最新、真实信息
  2. 数据库 / API 查询
    • 数字、订单、库存、价格：必须查库，不许编
  3. 计算器 / 代码解释器
    • 数学、统计、逻辑计算：交给工具，不依赖模型推理

五、后校验：自动检测 + 人工审核（必做）

  1. 自动事实校验（Fact-Checking）
  • 用另一个更强模型做 “审稿人”
  • 检查：
    • 是否存在不存在实体
    • 数据是否矛盾
    • 逻辑是否自洽
    • 能否被原文支持
  1. 关键词拦截

拦截幻觉高频句式：

  • “据相关资料显示…”
  • “专家普遍认为…”
  • “有研究表明…”（无具体来源）
  1. 人工审核流程
  • 高风险场景（医疗、法律、金融、新闻）必须人工终审
  • 建立错误样本库，持续迭代

六、按场景给你直接可用方案（最实用）

  1. 客服 / 问答（最容易幻觉）
  • 必须用 RAG
  • 禁止模型自由回答
  • 超出知识库统一回复：该问题我无法准确回答，将转接人工客服。
  1. 写作 / 总结
  • 要求每段事实必须来自原文
  • 禁止添加原文没有的信息、评价、数据
  1. 代码 / 技术
  • 代码必须可运行，给出验证方式
  • 技术参数必须来自官方文档（RAG）
  1. 医疗 / 法律 / 金融（强风险）
  • 模型只做信息整理，不做结论
  • 必须加免责声明
  • 关键结论必须人工复核

七、反幻觉终极 Prompt

你现在执行严格事实约束：
1. 不确定、无把握、无来源的内容，直接说“我无法确定”，绝不编造。
2. 不生成不存在的人名、地名、数据、论文、法规、链接、案例。
3. 所有事实性内容必须可验证、有权威依据。
4. 若问题需要最新信息或真实数据，你必须声明需要检索/查询，不能凭记忆回答。
5. 回答完成后，自我检查一遍是否存在幻觉，如有则删除错误内容。

总结（一句话记住）

  • 低风险场景：Prompt + 低温
  • 中风险场景：RAG + 来源引用
  • 高风险场景：RAG + 工具调用 + 自动校验 + 人工审核

AI 助手交互规范(ToB)

适用范围

本文档适用于产品内所有 AI 对话、AI 问答、AI 生成类功能的交互逻辑、输出约束、异常处理、用户体验规范。

设计原则

  1. 可信优先：事实准确 > 回答完整 > 体验流畅。
  2. 边界清晰：不超范围、不瞎编、不承诺、不联想。
  3. 可解释：所有关键信息必须可溯源、可校验。
  4. 可降级：AI 不可靠时，必须自动切换为人工 / 标准话术。
  5. 安全合规：禁止输出违法、违规、误导、承诺性内容。

输入规范

允许输入

  • 产品功能咨询、使用问题、操作指引
  • 规则说明、流程解释、状态查询
  • 合理的意见反馈、求助

禁止输入处理（触发拦截）

满足以下任一条件，直接拦截，不进入 AI 生成：

  • 涉政、色情、暴力、违法违规
  • 医疗诊断、法律意见、投资理财建议
  • 人身攻击、辱骂、恶意诱导
  • 非本产品相关的无关问题
  • 诱导泄露隐私、后台数据、内部逻辑

拦截统一回复：

抱歉，该问题我无法回答，请更换问题或咨询人工客服。

输出强制规范（核心反幻觉）

绝对禁止输出

  • 编造不存在的功能、价格、活动、规则、时间节点
  • 编造人名、数据、案例、文献、链接、政策依据
  • 做出承诺、保证、预判、结果担保
  • 超出知识库范围的联想、推理、扩展
  • 内部逻辑、接口、模型、数据结构等涉密信息

必须遵守

  1. 只依据官方知识库回答，无知识库内容则拒答。
  2. 不确定 → 明确说 “我无法确定”，不模糊、不猜测。
  3. 同类问题输出保持一致性，不可前后矛盾。
  4. 回答简洁、客观、中性，不情绪化、不夸张。

知识库与 RAG 流程（强制）

  1. 用户提问 → 先检索知识库。
  2. 匹配度≥阈值 → 基于片段生成回答。
  3. 匹配度＜阈值 → 直接拒答，不自由生成。
  4. 回答末尾必须显示：

信息来源于官方知识库

模型参数约束（给技术）

  • 生成模式：事实优先
  • 温度：≤ 0.2
  • 最大长度：根据场景限制，不允许超长发挥
  • 禁止开启：联想模式、扩展模式、创意模式

交互流程规范

正常流程

用户提问 → 意图识别 → 规则拦截 → 知识库检索 → AI 生成 → 校验 → 输出

降级流程

任一环节不通过，直接降级：

  • 无匹配知识 → 拒答
  • 校验不通过 → 拒答
  • 服务异常 → 提示稍后再试

降级统一回复：

抱歉，我暂时无法准确回答你的问题，你可以尝试咨询人工客服。

视觉与体验规范

  1. AI 回答区域必须明显区分用户输入，标明 “智能助手”。
  2. 关键步骤、操作指引支持高亮 / 可复制。
  3. 长回答分段展示，不堆砌大段文字。
  4. 禁止使用：“可能”“应该”“大概”“据了解” 等模糊表述。

异常处理

AI 服务异常

智能助手暂时无法服务，请稍后重试或联系客服。

重复提问（3 次以上同类无效问题）

我已经无法提供更多有效信息，建议你咨询人工客服获取帮助。

用户质疑答案错误

如果你认为信息有误，以官方页面 / 人工客服告知为准。你可以通过反馈提交问题。

日志与回溯（必接）

必须记录：

  • 用户提问（脱敏）

  • AI 回答

  • 检索到的知识库片段

  • 时间、设备、用户 ID（脱敏）

    用于：

  • 幻觉排查

  • 知识库优化

  • 客诉举证

审核与上线标准

AI 功能上线必须满足：

  • 有知识库，且内容准确最新
  • 有规则拦截
  • 有输出校验
  • 有降级话术
  • 有幻觉案例测试
  • 有免责声明
  • 有反馈入口

免责声明（必须展示）

本智能助手仅提供官方信息参考，不构成任何承诺或保证。具体以产品页面、官方规则及人工客服告知为准。

AI 助手交互规范（ToC）

文档目的

规范 AI 助手的输入、输出、边界、风控、幻觉规避、体验一致性，保证用户安全、可信、稳定使用，避免误导、编造、超范围回答。

适用范围

适用于产品内所有 AI 对话、智能问答、AI 辅助功能。

设计原则

  1. 安全第一：不违法、不违规、不误导、不承诺。
  2. 可信优先：事实准确 > 完整 > 流畅。
  3. 边界清晰：只做产品内允许的事，不自由发挥。
  4. 可降级：AI 不确定 / 不可靠时，必须兜底。
  5. 用户友好：口语化、简短、不生硬、不甩锅。

输入规则（用户提问）

允许处理

  • 产品功能使用咨询
  • 操作指引、步骤说明
  • 账号 / 权限 / 常见问题
  • 合理求助与反馈

强制拦截（不进入 AI）

以下内容直接拒绝，统一回复：

抱歉，这个问题我无法回答，请换个问题或咨询人工客服。

拦截范围：

  • 涉政、色情、暴力、恐怖、违法违规
  • 医疗诊断、法律建议、投资理财指导
  • 算命、占卜、玄学、人身攻击
  • 非本产品相关的外部问题
  • 诱导泄露内部数据、接口、逻辑
  • 诱导越狱、破解、违规操作

输出强制规则（反幻觉核心）

绝对禁止输出

  • 编造不存在的功能、活动、价格、福利、时间
  • 编造数据、案例、人名、链接、政策依据
  • 做出承诺、保证、预判、效果担保
  • 超出知识库 / 产品范围联想、推理、扩展
  • 使用 “可能”“应该”“大概”“据传闻” 等模糊表述
  • 引导用户违规、绕过限制

必须遵守

  1. 只依据官方知识库回答，无匹配则拒答。
  2. 不确定 → 明确说：“我无法确定”。
  3. 同类问题回答必须一致。
  4. 语言简短、口语化、正向，符合 ToC 用户习惯。
  5. 不使用专业术语、不堆砌长文本。

知识库与 RAG 强制流程（ToC 必开）

  1. 用户提问 → 意图识别 → 规则拦截
  2. 检索官方知识库
  3. 匹配达标 → 基于片段生成回答
  4. 匹配不达标 → 直接拒答，不自由生成
  5. 回答末尾统一标注：

信息以产品内实际展示为准

模型生成参数（给技术）

  • 模式：事实问答 / 保守模式
  • temperature ≤ 0.2
  • top_p ≤ 0.3
  • 禁止创意生成、自由联想、扩展发挥

交互流程

正常流程

用户提问 → 拦截检查 → 知识库检索 → AI 生成 → 校验 → 输出

降级流程（任一触发）

  • 无知识库匹配
  • 输出校验不通过
  • 服务异常 / 超时
  • 连续多轮无效回答

降级统一回复：

抱歉，我暂时无法准确回答，你可以咨询人工客服。

视觉与体验规范

  1. AI 回答区域明确标注 “智能助手”，与用户输入区分。
  2. 操作步骤分段、序号化、可复制。
  3. 关键信息（按钮、路径、入口）高亮突出。
  4. 不输出超长段落，不刷屏。

异常场景话术（固定）

  1. AI 服务异常

智能助手暂时无法使用，请稍后再试。

  1. 重复无效提问（≥3 次）

我暂时无法提供更多帮助，建议你咨询人工客服。

  1. 用户质疑答案错误

信息以产品内实际展示和官方规则为准，感谢你的反馈。

日志与回溯（必接）

必须记录（脱敏）：

  • 用户提问

  • AI 回答

  • 知识库匹配片段

  • 时间、设备、用户 ID（脱敏）

    用途：幻觉排查、体验优化、客诉处理。

上线验收标准

  • 有官方知识库并定期更新
  • 有输入拦截 & 输出校验
  • 有完整降级话术
  • 完成幻觉用例测试
  • 有用户反馈入口
  • 加免责声明

免责声明（页面固定展示）

本智能助手仅提供产品使用参考，不构成承诺或保证。

具体以产品实际页面、官方规则及人工客服说明为准。
